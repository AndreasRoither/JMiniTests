# T02 — Set project conventions

## Summary
Document and implement project-wide structure and naming conventions.

## Goals
- Standardize folders, naming, and shared types
- Reduce inconsistency before plugin growth

## Scope
- Define folder structure
- Define naming rules for content IDs and plugin IDs
- Define shared TypeScript types location
- Add linting and formatting rules
- Add architecture notes

## Deliverables
- Contributor-facing conventions note
- Stable top-level folder layout
- Shared naming rules

## Acceptance Criteria
- IDs are unique by convention
- Shared types have a single home
- New content/plugins can follow documented structure
